# Scope-Of-Work Layout

## Section Sequence

1. Hero banner
2. Summary table
3. At-a-glance block
4. `Phase 1 | Immediate blockers`
5. Numbered core workstreams
6. `Phase 2 | Nice-to-have follow-on work`
7. Closing matrix with:
   - recommended implementation order
   - definition of done

## Recommended Section Types

- Recruiter or access guardrails
- Needs-attention scoping
- Document ownership and downstream processing
- Product or schema decisions that are not simple bug fixes
- Lower-priority follow-on work

## Writing Rules

- Use short framing paragraphs before bullets.
- Prefer concrete claims over soft roadmap language.
- Keep bullets flat and outcome-oriented.
- Use the closing matrix to summarize implementation order rather than repeating that order in every section.
